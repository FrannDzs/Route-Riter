ComponentOne Query Schema XML Format

This document describes C1Query schema format. The XML document structure is presented here in a form similar to DTD document (Document Type Definition), but it is not a valid DTD and can not be used to verify a C1Query XML schema.

Clicking a hyperlink provides additional description of an element.

Legend:

#PCDATA - means that the element (node) has data. All data nodes contain strings or data converted to strings. Boolean values are represented by 0 (False) and 1 (True). Enumerated types are also represented by integer values.

(ACHILD?, ACHILD, ACHILD*) - if an element contains child nodes (elements) instead of data, the list of child nodes is provided.

? - means that the element is optional.
*- means that there can be 0 or more elements of this type.
If there is no ? or * qualifier, it means there is exactly one elements of this type.

Schema structure:

<!DOCTYPE SCHEMA [
<!ELEMENT CP (#PCDATA)>
<!ELEMENT IMPORTEDFROM (#PCDATA)>
<!ELEMENT SCHEMA (VIEWS, JOINRELATIONS?, CONDRELATIONS?, FOLDER, EXTENSIONS?, CATALOGS)>
    <!ELEMENT VIEWS (VIEW*)>
        <!ELEMENT VIEW (VIEWNAME, JOINGROUPTYPE?, CUSTOMWHERE?, VIEWFIELDS, VIEWTABLES)>
            <!ELEMENT VIEWNAME (#PCDATA)>
            <!ELEMENT JOINGROUPTYPE (#PCDATA)>
            <!ELEMENT CUSTOMWHERE (#PCDATA)>
            <!ELEMENT VIEWFIELDS (VIEWFIELD*)>
                <!ELEMENT VIEWFIELD (FIELDTABLENAME, FIELDNAME, FIELDTYPE, FIELDSIZE, FIELDPREC?, FIELDFORMULA?, FIELDLEFTJOINGROUPNAME?, FIELDRIGHTJOINGROUPNAME?)>
            <!ELEMENT FIELDTABLENAME (#PCDATA)>
            <!ELEMENT FIELDNAME (#PCDATA)>
            <!ELEMENT FIELDTYPE (#PCDATA)>
            <!ELEMENT FIELDSIZE (#PCDATA)>
            <!ELEMENT FIELDPREC   (#PCDATA)>
            <!ELEMENT FIELDFORMULA (#PCDATA)>
            <!ELEMENT FIELDLEFTJOINGROUPNAME (#PCDATA)>
            <!ELEMENT FIELDRIGHTJOINGROUPNAME (#PCDATA)>
            <!ELEMENT VIEWTABLES (VIEWTABLE*)>
                <!ELEMENT VIEWTABLE (TABLENAME, REQUIRED, JOINS?)>
                    <!ELEMENT TABLENAME (#PCDATA)>
                    <!ELEMENT REQUIRED (#PCDATA)>
                    <!ELEMENT JOINS (JOINTYPE, CONDITIONS)>
                        <!ELEMENT JOINTYPE (#PCDATA)>
                        <!ELEMENT CONDITIONS (CONDITION*)>
                            <!ELEMENT CONDITION (FIELD1?, TABLE1?, OPERATOR?, FIELD2?, TABLE2?, EXPRESSION?, USEEXPRESSION?, CONDITION?, USECONDITION?)>
                            <!ELEMENT FIELD1 (#PCDATA)>
                            <!ELEMENT TABLE1 (#PCDATA)>
                            <!ELEMENT OPERATOR (#PCDATA)>
                            <!ELEMENT FIELD2 (#PCDATA)>
                            <!ELEMENT TABLE2 (#PCDATA)>
                            <!ELEMENT EXPRESSION (#PCDATA)>
                            <!ELEMENT USEEXPRESSION (#PCDATA)>
                            <!ELEMENT CONDITION (#PCDATA)>
                            <!ELEMENT USECONDITION (#PCDATA)>
    <!ELEMENT JOINRELATIONS (JOINRELATION*)>
        <!ELEMENT JOINRELATION (RELATEVIEWS, VIEWLINKS, JOINS)>
            <!ELEMENT RELATEVIEWS (VIEWNAME*)>
                <!ELEMENT VIEWNAME (#PCDATA)>
            <!ELEMENT VIEWLINKS (VIEWLINK*)>
                <!ELEMENT VIEWLINK (VIEW1, JOINTYPE, VIEW2)>
                    <!ELEMENT VIEW1 (#PCDATA)>
                    <!ELEMENT JOINTYPE (#PCDATA)>
                    <!ELEMENT VIEW2 (#PCDATA)>
            <!ELEMENT JOINS (JOIN*)>
                <!ELEMENT JOIN (FIELD1?, TABLE1?, VIEW1?, OPERATOR?, FIELD2?, TABLE2?, VIEW2?, EXPRESSION?, USEEXPRESSION?, CONDITION?, USECONDITION?)>
                    <!ELEMENT FIELD1 (#PCDATA)>
                    <!ELEMENT TABLE1 (#PCDATA)>
                    <!ELEMENT VIEW1 (#PCDATA)>
                    <!ELEMENT OPERATOR (#PCDATA)>
                    <!ELEMENT FIELD2 (#PCDATA)>
                    <!ELEMENT TABLE2 (#PCDATA)>
                    <!ELEMENT VIEW2 (#PCDATA)>
                    <!ELEMENT EXPRESSION (#PCDATA)>
                    <!ELEMENT USEEXPRESSION (#PCDATA)>
                    <!ELEMENT CONDITION (#PCDATA)>
                    <!ELEMENT USECONDITION (#PCDATA)>
    <!ELEMENT CONDRELATIONS (CONDRELATION*)>
        <!ELEMENT CONDRELATION (RELATEVIEWS, CONDITION)>
            <!ELEMENT RELATEVIEWS (VIEWNAME*)>
                <!ELEMENT VIEWNAME (#PCDATA)>
            <!ELEMENT CONDITION (CONDITION*)>
    <!ELEMENT FOLDER (FOLDERNAME, SUBFOLDERS?, FOLDERFIELDS?)>
       <!ELEMENT FOLDERNAME (#PCDATA)>
        <!ELEMENT SUBFOLDERS (FOLDER*)>
        <!ELEMENT FOLDERFIELDS (FOLDERFIELD*)>
            <!ELEMENT FOLDERFIELD (DISPLAYNAME?, ALIASNAME?, TAG?, VIEW, NAME, TABLENAME, USESLOOKUPS?, LOOKUPVALUES?)>
            <!ELEMENT DISPLAYNAME (#PCDATA)>
            <!ELEMENT ALIASNAME (#PCDATA)>
            <!ELEMENT TAG (#PCDATA)>
            <!ELEMENT VIEW (#PCDATA)>
            <!ELEMENT NAME (#PCDATA)>
            <!ELEMENT TABLENAME (#PCDATA)>
            <!ELEMENT USESLOOKUPS (#PCDATA)>
            <!ELEMENT LOOKUPVALUES (VALUE*)>
                <!ELEMENT VALUE (USDATA, CAPTION)>
                    <!ELEMENT USDATA (#PCDATA)>
                    <!ELEMENT CAPTION (#PCDATA)>
    <!ELEMENT EXTENSIONS (EXTENSION*)>
        <!ELEMENT EXTENSION (EXTENDED, EXTENDS)>
            <!ELEMENT EXTENDED (VIEWNAME*)>
                <!ELEMENT VIEWNAME (#PCDATA)>
            <!ELEMENT EXTENDS (VIEWNAME*)>
                <!ELEMENT VIEWNAME (#PCDATA)>
    <!ELEMENT CATALOGS (CATALOG*)>
        <!ELEMENT CATALOG (VIEW*)>
            <!ELEMENT VIEW (#PCDATA)>
]>

 

<!ELEMENT SCHEMA (VIEWS, JOINRELATIONS?, CONDRELATIONS?, FOLDER, EXTENSIONS?, CATALOGS)>

Root element for schema data. Contains nested XML elements with all necessary information for views, folders, relationships and extensions.

<!ELEMENT CP (#PCDATA)>

Schema codepage. To set the codepage in the Schema Designer, use Schema->Options dialog->Schema tab->Schema codepage. Default value: 0 (neutral).

 Back to top

 

<!ELEMENT IMPORTEDFROM (#PCDATA)>

Contains OLE DB connection string from which the database schema was imported. This property is not used at run time, it is used only by Schema Designer when you connect to the database. This property is set in the Schema Designer >Options dialog->Schema tab->Connection String.

 Back to top

 

<!ELEMENT VIEWS (VIEW*)>

Contains the collection of schema views. Each view is represented by a VIEW element.

  Back to top

 

<!ELEMENT VIEW (VIEWNAME, JOINGROUPTYPE?, CUSTOMWHERE?, VIEWFIELDS, VIEWTABLES)>

Stores information about a schema view: view fields, view tables (tables the view consists of), relations between view tables.

  Back to top

 

<!ELEMENT VIEWNAME(#PCDATA)>

Contains view name.

  Back to top

 

<!ELEMENT JOINGROUPTYPE (#PCDATA)>

If you use join groups, specify here the type of the joins generated by the join groups of the view fields. Possible values:

0 - inner joins; 1 - outer joins.

This property is set in the Schema Designer in the "Use join groups as" combo box on the Advanced tab of the View Designer.

  Back to top

 

<!ELEMENT CUSTOMWHERE (#PCDATA)>

An optional SQL condition to be included in the WHERE clause of the generated SQL statement. Corresponding Schema Designer element: View Designer|Advanced tab|Custom WHERE condition.

  Back to top

 

<!ELEMENT VIEWFIELDS (VIEWFIELD*)>

The element storing view fields. Each view field is represented by a VIEWFIELD element.

Stores all necessary information about a view field.

<!ELEMENT FIELDTABLENAME (#PCDATA)>

Contains the view field's database table name.

  Back to top

 

<!ELEMENT FIELDNAME (#PCDATA)>

Contains the view field's database field name.

<!ELEMENT FIELDSIZE (#PCDATA)>

Contains the view field's size (integer value).

  Back to top


<!ELEMENT FIELDPREC   (#PCDATA)>

Contains the view field's precision (integer value).

<!ELEMENT FIELDFORMULA (#PCDATA)>

For a calculated field, this element contains the SQL expression that is used in the generated SQL statement. In this expression, use SQL syntax for functions and operators, and the following syntax for table fields: [table_name].[field_name]. For example:

[Order Details].[UnitPrice] * [Order Details].[Quantity] * (1 - [Order Details].[Discount])

<!ELEMENT FIELDLEFTJOINGROUPNAME (#PCDATA)>

Foreign join group name.

  Back to top


<!ELEMENT FIELDRIGHTJOINGROUPNAME (#PCDATA)>

Primary join group name.

<!ELEMENT VIEWTABLES (VIEWTABLE*)>

Contains nodes that contain all necesary information about view tables and relations between view tables.

  Back to top


<!ELEMENT VIEWTABLE (TABLENAME, REQUIRED, JOINS)>

Information about a table used in a view. A single table can be used in multiple views.

  Back to top


<!ELEMENT TABLENAME (#PCDATA)>

Contains view table name (a database table name).

  Back to top


<!ELEMENT REQUIRED (#PCDATA)>

Using this element, you can make a table optional, that is, included in the generated SQL statement only if one of its fields is directly specified or indirectly referenced in the user query. Possible values:  0 - not required, 1 - required.   Default value: required.

  Back to top

 

<!ELEMENT JOINS (JOINTYPE, CONDITIONS)>

Contains joins between this view table and view tables preceding it in the view.

  Back to top


<!ELEMENT JOINTYPE (#PCDATA)>

Defines join type for this table view. It corresponds to the Join type option located on the Table properties tab of the View Designer. Possible values:

0 - inner join, 1 - outer join, 2 - full outer join.

  Back to top


<!ELEMENT CONDITIONS (CONDITION*)>

Contains the list of join conditions used to join this view table with view tables preceding it in the view. Each condition corresponds to a CONDITION element.

  Back to top


<!ELEMENT CONDITION (FIELD1?, TABLE1?, OPERATOR?, FIELD2?, TABLE2?, EXPRESSION?, USEEXPRESSION?, CONDITION?, USECONDITION?)>

Join condition information (left/right fields and expressions). Note that all child elements are optional. This is because their use depends on the value of USECONDITION and USEEXPRESSION.

  Back to top


<!ELEMENT FIELD1 (#PCDATA)>

Left field name. This node is used in conjunction with TABLE1 node. If USECONDITION=1 (True, SQL condition is used), this node is ignored.

  Back to top


<!ELEMENT TABLE1 (#PCDATA)>

Left field table name. This node is used in conjunction with FIELD1 node. If USECONDITION=1 (True, SQL condition is used), this node is ignored.

  Back to top


<!ELEMENT OPERATOR (#PCDATA)>

The join condition's comparison operator. This node is used in conjunction with FIELD1 and FIELD2 or EXPRESSION nodes. Possible values:

0 - equal,
1 - Greater than,
2 - Less than,
3 - Greater or equal,
4 - Less or equal,
5 - Not equal,
6 - Is null (FIELD2 and EXPRESSION ignored),
7 - Is not null (FIELD2 and EXPRESSION ignored),
8 - Contains,
9 - Starts with.

  Back to top

 

<!ELEMENT FIELD2 (#PCDATA)>

Right field name. This node is used in conjuncion with TABLE2 node. This node is ignored if USECONDITION=1 (True).


  Back to top

 

<ELEMENT TABLE2 (#PCDATA)>

Right field table name. This node is used in conjunction with FIELD2 node. This node is ignored if USEEXPRESSION=1 (True).


 Back to top

 

<!ELEMENT EXPRESSION (#PCDATA)>

You can use SQL expression on the right part of a join condition. The following syntax is used for fields in the expression: [table_name].[field_name]. This value has effect only if USEEXPRESSION=1 (True) and USECONDITION=0 (False)

  Back to top


<!ELEMENT USEEXPRESSION (#PCDATA)>

Contains 1 if an an expression (EXPRESSION node) is used instead of a right side field. Contains 0 if a field (FIELD2 node) is used on the right side.

  Back to top


<!ELEMENT CONDITION (#PCDATA)>

The complete join condition is specified as an SQL text, without differentiating between its left and right side. The following syntax is used for fields in the condition:   [table_name].[field_name]. This node has effect only if USECONDITION=1 (True).

  Back to top


<!ELEMENT USECONDITION (#PCDATA)>

Contains 1 (True) if an SQL condition (CONDITION node) is used. If this node is set to 1, FIELD1,FIELD2 and EXPRESSION nodes are ignored. The SQL condition text is stored in the CONDITION node. In other cases, this node is set to 0 (False).

  Back to top


<!ELEMENT JOINRELATIONS (JOINRELATION*)>

Contains schema join relationships. Each join relationship is represented by a JOINRELATION element.

  Back to top


<!ELEMENT JOINRELATION (RELATEVIEWS, VIEWLINKS, JOINS)>

Contains all information about a join relationship: related views, join type used between these views, and join conditions.

  Back to top


<!ELEMENT RELATEVIEWS (VIEWNAME*)>

Contains a collection of VIEWNAME nodes enumerating the views participating in this join relationship.

  Back to top

 

<!ELEMENT VIEWNAME (#PCDATA)>

Contains a view name. Represents an individual node in the RELATEVIEWS collection.

  Back to top


<!ELEMENT VIEWLINKS (VIEWLINK*)>

Contains node that define how adjucent views are connected in the join relationship (join type). Each VIEWLINK element contains three subnodes: VIEW1, JOINTYPE and VIEW2. For any two consecutive VIEWLINK nodes, VIEW1 of the second must be equal to VIEW2 of the first. For example:

<VIEWLINK>
<VIEW1>Categories</VIEW1>
<JOINTYPE>0</JOINTYPE>
<VIEW2>Products</VIEW2>
</VIEWLINK>
<VIEWLINK>
<VIEW1>Products</VIEW1>
<JOINTYPE>1</JOINTYPE>
<VIEW2>Orders</VIEW2>
</VIEWLINK>

In a binary relationship, there is only one VIEWLINK element.

  Back to top


<!ELEMENT VIEWLINK (VIEW1, JOINTYPE, VIEW2)>

An individual node of the VIEWLINKS collection.

  Back to top


<!ELEMENT VIEW1 (#PCDATA)>     

Contains the view name of the first view in a VIEWLINK.

  Back to top

 

<!ELEMENT JOINTYPE (#PCDATA)>

Contains the type of join used in a VIEWLINK. The join connects the second view (VIEW2) to either the first view (VIEW1) or other views preceding the second view in the relationship (for a binary relationship, it is always the first view). For a binary relationship, this node corresponds to the using <join type> join combo box of the Join Relationship Designer. For an advanced (ternary or higher) relationship, this node corresponds to the Join type combo box.

Possible values for a binary relationship:

0 - inner;
1 - outer;
2 - full;
3 - two-way outer.

Possible values for a ternary or higher relationship:

0 - left inner;
1 - left outer;
2 - full outer.

  Back to top


<!ELEMENT VIEW2 (#PCDATA)>

Contains the view name of the second view in a VIEWLINK.

  Back to top


<!ELEMENT JOINS (JOIN*)>

Contains the joins (join conditions) of the join relationship. Each join is represented by a JOIN element.

  Back to top


<!ELEMENT JOIN (FIELD1?, TABLE1?, VIEW1?, OPERATOR?, FIELD2?, TABLE2?, VIEW2?, EXPRESSION?, USEEXPRESSION?, CONDITION?, USECONDITION?)>

Join condition information (left/right fields and expressions). Note that all child elements are optional. This is because their use depends on the value of USECONDITION and USEEXPRESSION.

  Back to top


<!ELEMENT FIELD1 (#PCDATA)>

Left field name. This node is used in conjunction with TABLE1 and VIEW1 nodes. If USECONDITION=1 (True, SQL condition is used), this node is ignored.

  Back to top


<!ELEMENT TABLE1 (#PCDATA)>

Left field table name. This node is used in conjunction with FIELD1 and VIEW1 nodes. If USECONDITION=1 (True, SQL condition is used), this node is ignored.

  Back to top


<!ELEMENT VIEW1 (#PCDATA)>

Left field view name. This node is used in conjunction with FIELD1 and TABLE1 nodes. If USECONDITION=1 (True, SQL condition is used), this node is ignored.

  Back to top


<!ELEMENT OPERATOR (#PCDATA)>

The join condition's comparison operator. Possible values:

0 - equal,
1 - Greater than,
2 - Less than,
3 - Greater or equal,
4 - Less or equal,
5 - Not equal,
6 - Is null (FIELD2 and EXPRESSION ignored),
7 - Is not null (FIELD2 and EXPRESSION ignored),
8 - Contains,
9 - Starts with.

  Back to top


<!ELEMENT FIELD2 (#PCDATA)>

Right field name. This node is used in conjunction with TABLE2 and VIEW2 nodes. This node is used only if both USEEXPRESSION and USECONDITION are False (0).

  Back to top


<!ELEMENT TABLE2 (#PCDATA)>

Right field table name. This node is used in conjunction with FIELD2 and VIEW2 nodes. This node is used only if both USEEXPRESSION and USECONDITION are False (0).

  Back to top


<!ELEMENT VIEW2 (#PCDATA)>

Right field view name. This node is used in conjunction with FIELD2 and TABLE2 nodes. This node is used only if both USEEXPRESSION and USECONDITION are False (0).

  Back to top


<!ELEMENT EXPRESSION (#PCDATA)>

You can use SQL expression on the right part of a join condition in a join relationship. An expression is used if USEEXPRESSION=1 (True) and USECONDITION=0 (False). The following syntax is used for fields in the expression:

[View name]![Table name].[Field name].

  Back to top


<!ELEMENT USEEXPRESSION (#PCDATA)>

Contains 1 if an an expression (EXPRESSION node) is used instead of a right side field. Contains 0 if a field (FIELD2/TABLE2/VIEW2 nodes) is used on the right side.

<!ELEMENT CONDITION (#PCDATA)>

The complete join condition is specified as an SQL text, without differentiating between its left and right side. This node has effect only if USECONDITION=1 (True). The following syntax is used for fields in the condition:

[View name]![Table name].[Field name].

  Back to top


<!ELEMENT USECONDITION (#PCDATA)>

Contains 1 (True) if an SQL condition (CONDITION node) is used. If this node is set to 1, FIELD1,FIELD2 and EXPRESSION nodes are ignored. The SQL condition text is stored in the CONDITION node. In other cases, this node is set to 0 (False).

<!ELEMENT USESLOOKUPS (#PCDATA)>

If this node is set to 1 (True), the end user can select a value for this folder field in a C1QueryFrame control from a predefined list of values. The list of values is specified in the LOOKUPVALUES node. Possible values:

0 (False): Lookup values are not used;

Contains the lookup list of values for this folder field.

<!ELEMENT VALUE (USDATA, CAPTION)>

A Value/Caption pair representing an element of the lookup list. The Value part is used when C1Query generates SQL statement. The Caption part is used to display this value in the C1QueryFrame control (both as the selected value in a query item and in the dropdown list where lookup values are presented for selection).

  Back to top


<!ELEMENT USDATA (#PCDATA)>

Contains the Value part of the Value/Caption pair. Each lookup value is stored as a string. This string is always encoded in English (United States) locale, regardless of the current regional settings. For example, if you are in Germany, you still need to specify numeric lookup values as "123.45" (in US format), not as "123,45". String values are stored in UTF-8 format

<!ELEMENT CAPTION (#PCDATA)>

Contains the Caption part of the Value/Caption pair. If the caption is empty, C1QueryFrame control displays the default string representation of the value.

<!ELEMENT CATALOGS (CATALOG*)>

This elements contains a collection of so called catalogs (also known as view groups). Catalogs are independent groups of views. All views in a group are connected with relationships or join groups, and no two views belonging to different groups are connected. A schema can contain a single view group, or multiple view groups. Each of them can be used (only one at a time) in a user query. The C1Query.MainViewName property determines which catalog is currently used. The default is the first catalog.

  Back to top
