Changes and Bug fixes for TDBList7 from build 7.0.0.269  to build
7.0.0.278					       10 JAN 2002
==================================================================


** TDBCombo bug(bug#5229), fixed
   When using the TDBCombo and AutoCompletion = true, if you
   backspace on a character in the text portion of the combo,
   it does not take you back to the previous value in the 
   dropdown list.

** TDBCombo bug(bug#5363), fixed
   When using AutoCompletion = true and the AutoDropDown
   = true, if you select something that is not in the list,
   leave the control, and the next selection is not coming
   up.

** TDBCombo bug(bug#5506), fixed
   SelectedItem is not working in Form_Load and TDBCombo_OnInit 
   events, although it is working in Form_Active event.

** TDBCombo bug(bug#5603), fixed
   Setting combo Text property under Form_Load and 
   TDBComco_OnInit does not work. This bug is related to
   bug#5506.

** TDBCombo bug(bug#5662), fixed
   Setting TDBCombo1.SelectedItem = TDBCombo1.SelectedItem
   triggers ItemChange event.

** TDBCombo bug(bug#5724), fixed
   BoundText does not get set properly.

** TDBList bug(bug#5735), fixed
   If you set the Bookmark or SelectedItem property for TDBList
   in the Form_Load event, this property appears to be reset back
   to null by the time the Form_Initialize event fires.

** TDBCombo bug(bug#5772), fixed
   ListField property is not working properly under Storage
   mode. We actually simplified the process a little in storage
   mode. You need not set the ListField and DataField of the
   column in the same time now; you only need to set the 
   ListField to the caption of the column.

** TDBCombo bug(bug#5804), fixed
   When setting the TDBCombo with a caption and one record, the
   dropdown list does not display correctly.

** TDBCombo bug(bug#5838), fixed
   In storage mode, it will not work if you set the SelectedItem
   to a value which is not initially visible in the combo 
   dropdown(in other words, those rows that you have to scroll 
   down the dropdown list to see).

** TDBCombo bug(bug#5239), fixed
   When you drop down below the bottom of the screen, the combo 
   is displayed an inch or two above the actual control.

** TDBCombo bug(bug#5415), fixed
   Setting a value in the Text portion of the TDBCombo on the
   Lost_Focus event is not being filled when the user clears the 
   first value out.

** TDBCombo bug(bug#5642), fixed
   If the form contains a VSElasticLight control, then the cursor
   will not appear in the text portion of the combo.

** TDBList bug(bug#5936), fixed
   When you select a row and click the heading of a column, then
   the row is unselected, but SelectedItem still returns a value.

** TDBList bug(bug#5977), fixed
   When MultiSelect is set to None, and you click the same row 
   multiple times, the SelBookmarks.Count property changes from 1
   to 0 back and forth.
