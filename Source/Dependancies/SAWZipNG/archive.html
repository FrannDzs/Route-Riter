Archive object

Properties

ArchivePath
Type String Read
Returns the path of the archive.

AutoFlush
Type Boolean Read/Write
When AutoFlush is true, the archive is flushed after each operation that changes the archive (adding, deleting and modifying the archive comment and file comment). For a big archive setting AutoFlush to true, can influence the performance.

CaseSensitivity
Type Boolean Read/Write
Set the case-sensitivity of FindFile and FindFiles.

Closed
Type Boolean Read
Returns true when the archive is closed. False when the archive is opened.

Comment
Type String Read/Write
Get/Set the comment of the archive.

Count
Type Long Read
Returns the number of files and directories in the archive. See also DirCount and FileCount.

CurrentDisk
Type Integer Read
Returns the current disk of the spanning archive.

DirCount
Type Long Read
Returns the number of directories the archive. See also Count and FileCount.

EnableFindFast
Type Boolean Read/Write
Enables fast finding by the file name of the files inside the archive. See FindFile and FindFiles.

FileCount
Type Long Read
Returns the number of files in the archive. See also Count and DirCount.

GeneralBufferSize
Type Long Read/Write
Get/Set the size of the general buffer. This buffer is used in extracting, compressing, deleting and renaming files. The minimum size is 1024. The default is 65536.

IgnoreCrc
Type Boolean Read/Write
When IgnoreCrc is set to true, CRC is not checked for the files being tested or extracted. This can be useful when working with Java TM Archives (jar). The CRC is checked by default.

Password
Type String Read/Write
When adding files the password will be used to encrypt the files. When extracting files, the password will be used to decrypt the files. Set the password before adding or extracting files. Use an empty String "" to stop encryption.

Readonly
Type Boolean Read
Returns true when the archive is readonly.

RootPath
Type String Read/Write
When the root path is set, this path will be omitted from the file that is added to the archive (Relative paths). Set this property before you start adding files. Use an empty String "" to use full paths again.

SearchBufferSize
Type Long Read/Write
Get/Set the size of the searchbuffer. This buffer is used to search for the central dir. The minimum size is 1024. The default is 32768.

SpanMode
Type SpanMode Read
Returns the span mode of the archive.

SystemCompatibility
Type ZipPlatform Read/Write
Get/Set the System compatibility. This property influences the file attribute conversion.

TempPath
Type String Read/Write
Get/Set the temporary path. This path is used when compressing files and SM_CHECK_FOR_EFF is used for smartness in AddFile and the disk spanning archive is in creation

WriteBufferSize
Type Long Read/Write
Get/Set the size of the writebuffer. This buffer is used to hold data before it is written to disk. The optimal size for the write buffer in the disk spanning archive is the size of the volume. The minimum size is 1024. The default is 65536.

FindFiles
Function FindFiles(Pattern As String, [fullpath As Boolean = True]) As Variant
Pattern A pattern string:
  • * matches any sequence of characters(zero or more)
  • ? matches any character
  • [SET] matches any character in the specified set,
  • [!SET] or[^SET] matches any character not in the specified set
    To suppress the special syntactic significance of any of []*?!^-\, and match the character exactly, precede it with a \
    The case-sensitivity of the pattern is set to CaseSensitivity.
    • filenameOnly When true, FindFile tries to find a filename without a path.
    This function finds the indexes of the files, which filenames match the specified pattern.

    Flush
    Sub Flush()
    Writes the central dir and the internal buffers to disk. Use it after modifying the archive if you want to prevent the loss of the data you've compressed so far in case of a program crash. When AutoFlush is true, you don't need to call this function.
